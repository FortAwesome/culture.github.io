I CAN HAS DOC?⌗

Based on discussions at the last snuggle, we discussed the need for additional documentation. I suggested using the GitHub Pages features as a way that we might improve our documenation and gain some cool features. To quote Mike I thought we could “feed two birds with one scone” and also document some of our company culture here instead of having to search for interesting culture discussions that happened in slack. To that end, here is a simple example of the GitHub Pages in use.

Basic Setup of the Site⌗

  1. Create a place to store the content, in this case, it is in the culture-awesome repo.
  2. Create a place to host the page, in this case, culture.github.io.
  3. Install Hugo.
  4. Pull down the culture-awesome repo and create a new site. 
    hugo new site culture-blog
  5. Find a dope ass theme (in this case we decided on terminal
  6. Clone the theme into the themes folder created when we created a new site 
    git clone https://github.com/panr/hugo-theme-terminal.git themes/terminal
  7. Update the config.toml file (I used the recommended settings here with a couple minor tweaks
  8. Create a simple post (the one your currently reading) with… 
    hugo new posts/i_can_has_doc.md
  9. Write a bunch of stuff in markdown.
  10. Validate everything is working. 
    hugo server
  11. Check it out by navigating to http://localhost:1313
  12. With the initial site setup we need to configure hosting

Basic Setup for Hosting⌗

  1. Pull down the culture.github.io repo
  2. We’ll make culture-blog a sub-module so we need a main branch and an initial commit (if you want to setup another one of these for yourself one day, otherwise, keep readin')
  3. Use the following command in the culture-blog folder of the culture-awesome repo. This will automatically add our statically generated pages to the GitHub Pages when you run the hugo build commands (again, already complete so just documenting for posterity) 
    git submodule add -b main git@github.com:FortAwesome/culture.github.io.git public
  4. Generate the static files using the name of your theme, in this case terminal 
    hugo -t terminal
  5. Hugo should report that several files and at least one sitemap were created. They should be visible in the /public directory. Additionally if you navigate to the directory and run the following command you should see that the public origin is pointed to our GitHub Page and not the culture-awesome repo 
    git remote -v
